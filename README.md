# ML Challenge – AlrIEEEna'26

This project was developed for the IEEE SB GEHU ML Challenge.

The goal is to predict whether a device is operating normally or experiencing a fault using 47 numerical sensor features.

Class labels:
- **0** → Normal
- **1** → Faulty

# LightGBM

LightGBM (Light Gradient Boosting Machine) is a gradient boosting framework that builds decision trees sequentially to improve prediction accuracy. Each new tree focuses on correcting the errors made by the previous trees. Compared to traditional boosting algorithms, LightGBM is optimized for speed and efficiency and can handle large feature spaces effectively. In this project, LightGBM was selected as the final model because it consistently achieved the highest F1-score among the tested algorithms.

# How to Run the Project

1. Clone the repository:

git clone https://github.com/get-gud05/AlrIEEEna26.git
cd AlrIEEEna26

2. Make sure the dataset files `TRAIN.csv` and `TEST.csv` are present in the project directory.

3. Open the notebook:

jupyter lab

or

jupyter notebook

4. Run all cells in **Model.ipynb**.

5. The notebook will:
- Train the LightGBM model
- Generate predictions for the test dataset
- Create the required output file `FINAL.csv`

---

# Model Used

Several machine learning models were evaluated:

- Random Forest
- Logistic Regression
- XGBoost
- CatBoost
- LightGBM

The final model selected was **LightGBM**, which achieved the best performance with an **F1-score of approximately 98.6–98.7**.

---

# Output Format

The final output file `FINAL.csv` has the following format:

ID,CLASS  
1,1  
2,0  
3,1  

Where:

- **ID** → identifier from the test dataset  
- **CLASS** → predicted label (0 = Normal, 1 = Faulty)
